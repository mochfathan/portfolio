# Marketing AI Workflow Console

A browser-based AI workflow tool for lead qualification, email drafting, creative brief generation, task routing, and audit logging. Supports multiple AI providers or runs fully offline in simulation mode — no installation required.

---

## Files in this bundle

```
index.html   — application interface
styles.css   — visual styling
app.js       — workflow logic and AI provider layer
README.md    — this file
```

All four files must be kept in the same directory for the application to work correctly.

---

## Quick start

1. Download all four files into the same folder on your computer.
2. Open `index.html` in any modern browser (Chrome, Firefox, Edge, Safari).
3. In **Step 0 — AI provider**, select a provider or leave it on **Simulation mode**.
4. Fill in lead details in **Step 1 — Lead intake**, or click **Load sample** to use the built-in example.
5. Click **Run workflow**.

No server, no installation, no account required to use simulation mode.

## Step-by-step walkthrough

### Step 0 — AI provider

This is the first panel at the top of the workspace. Configure it before running the workflow.

| Field | Description |
|---|---|
| Provider | Select Claude, OpenAI, Custom endpoint, or Simulation mode |
| Model | Auto-populated based on provider. Change if needed. |
| API key | Your personal API key from the provider. Stored in browser memory only — never sent anywhere except directly to the selected provider endpoint. |
| Custom endpoint URL | Visible only when Custom is selected. Enter the base URL of your in-house API (e.g. `https://your-api.example.com/v1`). Must accept OpenAI-compatible POST requests. |

The status indicator in the sidebar updates automatically:

- **Grey dot** — no provider configured, simulation mode active
- **Green dot** — API key entered, provider ready
- **Amber pulsing dot** — workflow is running

You can switch providers between runs without resetting the lead data.

---

### Step 1 — Lead intake

Enter the details of the inbound lead you want to process.

| Field | Description |
|---|---|
| Company | Name of the prospect's organisation |
| Contact name | Name of the person you will be writing to |
| Industry | Select the closest match from the dropdown |
| Goal | Free-text description of their business need, campaign objective, or pain point |

Click **Load sample** to populate all fields with a pre-built Healthcare example (Northstar Dental Group). Useful for first-time testing.

---

### Step 2 — Qualification

Populated automatically when you run the workflow. Displays:

- **Fit score** — 0 to 96, higher is a stronger match
- **Fit assessment** — one-sentence evaluation
- **Urgency** — High, Medium, or Low
- **Next action** — recommended immediate step
- **Risk note** — governance or compliance flag

In live AI mode, this is generated by the selected AI provider based on the lead data and your knowledge inputs. In simulation mode, it is calculated by a deterministic scoring engine built into the app.

---

### Step 3 — Project knowledge

These two fields act as the knowledge base the AI draws on when generating drafts. Edit them to match your actual business context before running.

| Field | Description |
|---|---|
| Brand voice | A short description of the tone and style your communications should follow |
| Approved offer | The specific service, package, or proposal you are authorised to present to prospects |

These fields persist across runs within the same session. Reset only clears lead intake fields, not knowledge fields.

---

### Step 4 — Workflow controls

Four toggles that govern what the workflow produces and how it behaves.

| Toggle | Default | Effect when on |
|---|---|---|
| Require human approval before sending | On | Adds a visible review status note to all email drafts |
| Include source notes in draft | On | Appends a provenance note listing what inputs were used to generate the draft |
| Create Notion-style follow-up task | On | Generates a task card in Step 7 with owner, status, and priority |
| Create Canva-style creative brief | On | Generates a design brief in Step 6 alongside the email draft |

---

### Step 5 — Email draft

Generated automatically after running the workflow. The draft is written to match the brand voice in Step 3 and addresses the specific goal entered in Step 1.

- Click **Copy** to copy the full draft to your clipboard.
- If human approval is enabled, the draft includes a visible status note at the bottom reminding the reviewer not to send without checking.
- In live AI mode, the content is generated by the selected provider. In simulation mode, a structured template is populated with the lead data.

---

### Step 6 — Creative brief

A design brief formatted for handoff to a creative team or Canva workflow. Covers objective, audience, message angle, tone, and industry-specific signals relevant to the lead's sector.

- Click **Copy** to copy to clipboard.
- Disable this output in Step 4 if you do not need a creative brief for the current run.

---

### Step 7 — Task routing

Two task cards are generated when task creation is enabled in Step 4:

1. **Follow up with [Company]** — includes priority level and recommended next action drawn from the qualification output.
2. **Request source materials** — a standing reminder to collect campaign exports, landing page URLs, and lead quality notes before the discovery call.

These are formatted to mirror Notion task cards and can be copy-pasted directly into a Notion database or any task management tool.

---

### Step 8 — SOP checklist

A fixed six-step standard operating procedure that governs every workflow run:

1. Capture lead data from approved source
2. Classify fit, urgency, risk, and recommended next action
3. Retrieve approved offer and brand voice
4. Generate draft email and internal task
5. Human reviewer approves before external sending
6. Log decision, owner, and next follow-up date

This checklist is intentionally static — it represents the governance standard the workflow is built around.

---

### Audit log

Every action taken during a workflow run is timestamped and recorded here, including which provider generated each output. The log reads newest-first. Entries include:

- Workflow start, lead intake, knowledge retrieval
- AI provider requests and responses
- Draft generation confirmations
- Governance gate status
- Any errors with descriptive messages

The audit log clears when you click **Reset**. If you need to retain a record, copy the log entries before resetting.

## Provider setup guides

### Claude (Anthropic)

1. Go to [console.anthropic.com](https://console.anthropic.com)
2. Sign in or create an account
3. Navigate to **API Keys** in the left sidebar
4. Click **Create Key**, give it a name, and copy the key
5. Paste it into the API key field in Step 0
6. Select **Claude Sonnet 4** as the model (recommended for balance of quality and speed)

### OpenAI

1. Go to [platform.openai.com](https://platform.openai.com)
2. Sign in or create an account
3. Navigate to **API Keys** under your account settings
4. Click **Create new secret key**, copy it immediately (it is only shown once)
5. Paste it into the API key field in Step 0
6. Select **GPT-4o mini** as the model (recommended for cost efficiency)

### Custom / in-house endpoint

1. Obtain your organisation's internal API base URL and access credentials from your IT or AI team
2. Select **Custom / In-house endpoint** in the provider dropdown
3. Enter the base URL (e.g. `https://ai-gateway.yourcompany.com/v1`) — do not include `/chat/completions` at the end, the app appends this automatically
4. Enter your access token or API key in the API key field
5. The app sends OpenAI-compatible POST requests. If your endpoint uses a different schema, a developer will need to adjust the request structure in `app.js` under the `callAI()` function

## Simulation mode

Select **Simulation mode** in Step 0 to run the full workflow without any API key. All outputs are generated by deterministic logic built into the app:

- Qualification score is calculated from keyword matching against the goal text
- Email draft is built from a structured template populated with lead data
- Creative brief uses industry-specific signal lists pre-loaded in the app

Simulation mode is useful for:
- Demonstrating the workflow structure to a client or team without consuming API credits
- Testing the form and governance controls
- Running offline without internet access

## Deployment on GitHub Pages

To share a live version of this tool:

1. Create a repository on [github.com](https://github.com)
2. Upload all four files (`index.html`, `styles.css`, `app.js`, `README.md`) to the repository root
3. Go to **Settings → Pages** in the repository
4. Under **Source**, select **Deploy from a branch**, choose `main`, and save
5. GitHub will publish the site at `https://yourusername.github.io/repository-name/`

The README renders automatically on the repository homepage. The live app is accessible at the Pages URL.

## Security notes

- API keys are stored in browser memory for the duration of the session only. They are not written to localStorage, cookies, or any external service.
- Keys are transmitted directly to the selected provider endpoint using HTTPS. They do not pass through any intermediate server.
- Closing or refreshing the browser tab clears the key from memory. You will need to re-enter it in the next session.
- Do not share your API key with others or commit it to a public repository.

## Troubleshooting

**The workflow runs but outputs look like templates, not AI-generated text.**
You are in simulation mode. Check that a live provider is selected in Step 0 and that a valid API key is entered.

**Error: API key is required.**
You selected a live provider but left the API key field empty. Enter your key in Step 0.

**Error: [provider] API error (401).**
Your API key is invalid or has been revoked. Generate a new key from the provider console and re-enter it.

**Error: [provider] API error (429).**
You have exceeded your API rate limit or run out of credits. Check your usage in the provider console.

**Error: Custom endpoint — enter your custom endpoint URL.**
You selected Custom provider but left the endpoint URL field blank. Enter your organisation's API base URL.

**The creative brief says "Creative brief creation is turned off."**
The Canva-style creative brief toggle in Step 4 is unchecked. Enable it and run the workflow again.

**Outputs are blank after switching providers mid-session.**
This is expected. Switching providers does not re-run the workflow. Click **Run workflow** again to regenerate outputs with the new provider.
